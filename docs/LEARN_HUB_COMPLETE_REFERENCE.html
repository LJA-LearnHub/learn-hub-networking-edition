Part I — Introduction

What this project is

Learn Hub is a client-only learning application: one index.html loads several large JavaScript bundles that define hundreds of lessons across eight tracks (HTML, CSS, JavaScript, Python, SQL, Security (workplace / Security+ prep), Labs (Kali VM hands-on curriculum), and CompTIA Tech+). There is no backend; progress and preferences live in localStorage. Practice tracks embed live editors (HTML/CSS/JS in a sandboxed iframe, Python via Pyodide, SQL via sql.js). Tech+ adds long-form study-guide content from Markdown, split into lazy-loaded chapter chunks, plus imported GimKit question-set bundles for Tech+ Questions mode. Security and Labs ship similar long-form HTML from Markdown-derived bundles (learn-hub-security-md.js, learn-hub-kali-md.js) merged at boot.

Runtime architecture

The DOM is a fixed shell: left sidebar (course pills, XP bar, filterable lesson list, accessibility toggles, reset), center main (top bar with title, lesson picker, progress bar, optional “Hide This step”), and a two-column content grid — on wide screens, Practice (editors / quiz) left and This step / Notes / Reading right. On small screens the layout stacks; Tech+ “learn” steps hide the practice column entirely (learn-only body class).

Part II — How the app runs

Boot sequence & script load order

index.html loads scripts in this order:

  1. learn-hub-courses.js → window.LEARN_HUB_COURSES
  2. learn-hub-depth.js → window.LEARN_HUB_DEPTH (reserved; often empty)
  3. learn-hub-techplus.js → window.LEARN_HUB_TECHPLUS
  4. learn-hub-gimkit-questions.js → window.LEARN_HUB_GIMKIT_QUIZZES (generated question sets for Tech+ Questions mode)
  5. learn-hub-techplus-md.js → loader, hashes, loadLearnHubTechplusChapter(n)
  6. learn-hub-security-md.js → window.LEARN_HUB_SECURITY_MD (Security track HTML segments)
  7. learn-hub-kali-md.js → window.LEARN_HUB_KALI_MD (Labs / Kali curriculum segments)
  8. learn-hub-deep.js → window.LEARN_HUB_DEEP
  9. learn-hub-playground.js → window.LEARN_HUB_PLAYGROUND
  10. learn-hub-app.js → merges curriculum, binds UI, learnHubRunApp on DOM ready

Immediately after the first IIFE in learn-hub-app.js, each lesson’s narrative is concatenated with LEARN_HUB_DEPTH[id], LEARN_HUB_DEEP[id], LEARN_HUB_TECHPLUS[id] → readHtmlBase; then LEARN_HUB_TECHPLUS_MD[id], LEARN_HUB_SECURITY_MD[id], and LEARN_HUB_KALI_MD[id] are appended (in that order) into readHtml; narrative is cleared. On failure, a minimal “Setup” course is injected.

Curriculum model & merge rules

Course: id, name, ws (web | py | sql | tech for Tech+, Security, and Labs — reading/quiz layout without web/py/sql editors), lessons[].

Lesson: id, unit (sidebar grouping), kind (learn | practice | challenge | quiz), title, optional narrative (usually empty in bundle), starter / starterPy / starterSql, seed (SQL), check (autograder), questions (Tech+ MCQ).

Tech+ study-guide ids: tech-sg-NN-MM (chapter NN, segment MM). Reading often loads from LEARN_HUB_TECHPLUS_MD after loadLearnHubTechplusChapter(N).

Locking: Courses whose ws is "tech" (Tech+, Security, Labs) have no linear lock — any lesson can be opened anytime. All other workspaces (web, py, sql) require the previous lesson in that course to be marked done before the next unlocks.

LEARN_HUB_DEEP, Tech+, Security, and Labs bundles

Objects keyed by lesson id → HTML string. Merge order into readable content: LEARN_HUB_DEPTH → LEARN_HUB_DEEP → LEARN_HUB_TECHPLUS as readHtmlBase, then LEARN_HUB_TECHPLUS_MD, LEARN_HUB_SECURITY_MD, and LEARN_HUB_KALI_MD are appended in order for readHtml.

  • DEEP — Long-form reference for web/css/js/py/sql tracks (MDN links, recurring footers, callouts).
  • TECHPLUS — Shorter Tech+ excerpts for lessons that are not full MD segments (intros, etc.).
  • TECHPLUS_MD — Large study-guide segments for tech-sg-*; populated by lazy-loaded learn-hub-techplus-md-chNN.js files.
  • SECURITY_MD — Segments for lh-sec-* and Security quizzes; built from docs/SECURITY_CONCEPTS_WORKPLACE_CURRICULUM.md (npm run build:security).
  • KALI_MD — Segments for labs-kali-*; built from docs/KALI_HANDS_ON_LAB_CURRICULUM.md (npm run build:kali). Optional npm run emit:kali-slow overwrites that Markdown from emit-kali-slow-curriculum.mjs, which imports phased lab prose from kali-lab-rich-procedures.mjs.

Build pipeline (Markdown → chunks)

scripts/build-techplus-from-markdown.mjs:

  1. CHAPTER_FILES — 12 rows [tech-sg-NN, filename, chapter title].
  2. preprocessMarkdown — PDF cleanup (tables, URLs, hyphens, …).
  3. marked.parse — { gfm: true, breaks: false }.
  4. polishTechplusHtml — mojibake, exercise repairs, OCR words, “see figure” text, etc.
  5. findHtmlSplitMarkers — <h3, EXERCISE N.M, Review Questions; dedupe by MIN_MARKER_GAP (48).
  6. slice — chunks ≥ 80 chars; oversize split at </p> (max 16500); tiny merge (<120) unless lead <h3.
  7. Ids — tech-sg-NN-MM per segment; segmentLessonTitle for sidebar.
  8. Output — writeChapterJsFiles + writeTechplusMdLoader; may update learn-hub-courses.js.

Part VI — Reference

Edge cases & defensive behavior

  • Curriculum parse failure → “Setup” course.
  • Boot exception → error in teach; __LH_BOOT_OK watchdog @ 3s.
  • Locked index → jump to first unlocked (non-tech).
  • Tech+ filter: ch N, chapter keywords, bare 1–12.
  • Chapter search: occurrence-based; modulo navigation.
  • SQL CDN fail → error panel; other tracks OK.
  • Build uses marked as library (not CLI) on Windows.

Large generated bundles

learn-hub-courses.js, learn-hub-deep.js, learn-hub-techplus.js, learn-hub-security-md.js, learn-hub-kali-md.js, and each learn-hub-techplus-md-chNN.js are data-dense (often one very long line). Document their schema (Parts II & IV) and edit sources (chapters/*.md, Security/Kali Markdown, generators) rather than hand-editing minified output unless you intend to.
